# 피싱 인식 실습 · 가짜 로그인 페이지의 정체

> 가상 브랜드 **SkyBird**(실제로 존재하지 않는 SNS)의 가짜 로그인 페이지를 직접 체험합니다.
> 학생이 정보를 입력하는 순간 **"이건 피싱이었다"** 를 폭로하고, 어떻게 알아챘어야 하는지 가르칩니다.
>
> 🔒 입력값은 외부로 전송되지 않고, 비밀번호는 가려서 표시하며, 종료 시 모두 삭제됩니다.

## 0. 시작하기 (강사)

1. **[▶ 시작]** 버튼 → 상태 **실행 중** 확인.
2. 화면 2개를 띄워둡니다:
   - **학생용 가짜 로그인:** `http://localhost:5001` (학생은 LAN 접속 주소 사용)
   - **강사용 "공격자 대시보드":** `http://localhost:5001/instructor` (강사 화면/빔프로젝터)

## 1. 함정 깔기 (5분)

- 학생들에게 평범하게 안내합니다: *"SkyBird라는 새 SNS 로그인 화면이래. 한번 로그인해볼래?"*
- **진짜 비밀번호는 절대 쓰지 말고, 아무 가짜 값**을 넣으라고 미리 안내하세요. (중요)
- 학생들이 가짜 아이디/비밀번호를 입력하고 로그인 버튼을 누름.

## 2. 폭로 (10분)

- 로그인을 누르는 순간 학생 화면에 **"🎣 방금 당신은 피싱에 당했습니다"** 가 뜹니다.
- 동시에 강사 화면(`/instructor`)에는 학생들이 넣은 값이 실시간으로 쌓입니다.
- 빔프로젝터로 보여주며: *"가짜 페이지 하나로 공격자는 이렇게 정보를 모읍니다."*

## 3. 식별법 토론 (15분)

폭로 화면에 나온 단서들을 함께 짚습니다:

| 단서 | 확인 방법 |
|---|---|
| **도메인(주소창)** | 진짜 주소가 맞나? 비슷한 철자로 속이지 않나? (`g00gle` 등) |
| **HTTPS / 자물쇠** | 로그인 페이지가 `http`면 위험 |
| **급한 미끼** | "지금 로그인하면 선물" 같은 압박은 전형적 수법 |
| **유입 경로** | 메시지/메일 링크로 온 로그인 창은 특히 의심, 직접 주소 입력 |
| **비밀번호 재사용** | 한 곳이 털리면 전부 위험 |

## 4. 마무리 (10분)

- 학생들이 평소 쓰는 서비스(메신저, 게임 계정 등)에 어떻게 적용할지 이야기.
- "이상하면 입력하지 말고, 직접 공식 앱/사이트로 들어가 확인하기" 습관 강조.
- **허락 없이 타인을 대상으로 가짜 페이지를 만들면 범죄**임을 분명히 합니다.

---

### 강사 메모
- 학생이 동시에 접속해도 됩니다(멀티). 각 입력은 강사 화면에 모입니다.
- 기록은 메모리에 최근 20건만 보관 → **종료 후 시작**하면 초기화.
- 이 실습의 목표는 '공격 기술'이 아니라 **'속지 않는 눈'** 을 기르는 것입니다.
